# Debugger 5 分钟接线指南

本文面向首次接线 `RDC-Agent-Frameworks/debugger` 的维护者或高级用户，目标是在 5 分钟内完成一条可验证的手工接线。

## 1. 选择宿主模板

从 `debugger/platforms/<platform>/` 中选择目标宿主模板，例如 `codex`、`codex_plugin`、`claude-code`、`cursor`。

补充说明：

- `codex` 直接使用 `debugger/platforms/codex/`。
- `codex_plugin` 的外层目录是 `debugger/platforms/codex_plugin/`，真正需要接线和同步的 plugin root 是其中的 `debugger/platforms/codex_plugin/rdc-debugger/`。

## 2. 覆盖共享正文

将 `debugger/common/` 整体覆盖到复制后的目标平台包根目录的 `common/`。

## 3. 覆盖 package-local `tools/`

将 `RDC-Agent-Tools` 根目录整包拷贝到复制后的目标平台包根目录的 `tools/`，覆盖占位内容。

注意：

- `platform_adapter.json` 中的 `paths.tools_source_root` 固定为 `tools`；不要再把它当成手工绑定入口或 live runtime 目录。
- source repo 中平台模板的 `tools/` 目录只保留最小占位内容；不要直接把 source root 当成可运行平台包。

## 4. 运行接线校验

在复制后的平台包根目录执行：

```bat
python common/config/validate_binding.py --strict
```

通过标准：

- package-local `tools/` 不是占位目录。
- `validation.required_paths` 在 `tools/` 下全部存在。
- `tool_catalog.snapshot.json` 与 `tools/spec/tool_catalog.json` 对齐。
- `tool_catalog.snapshot.json` 必须同步反映当前 `rdx-tools` 的 event-bound / replace / debug contract，例如 `resolved_event_id`、`replacement_id/status='applied'`、以及 `failure_stage` / `failure_reason` / `attempts`。
- 宿主入口文件和共享文档入口都存在。
- Claude Code 若启用 hooks，`matcher` 必须是 string；文件路径过滤由共享 hook dispatcher 根据 hook payload 判断，不写 object matcher。

## 5. 再进入宿主

接线通过后，再在目标宿主中打开模板根目录，并按 `debugger/README.md` 的流程提供 `.rdc` 后开始调试。

若当前选择的是 `codex_plugin`：

- 接线通过后，先将 `rdc-debugger/` 同步到 `~/.agents/plugins/rdc-debugger/`。
- 再把 `debugger/platforms/codex_plugin/references/personal-marketplace.sample.json` 合并到 `~/.agents/plugins/marketplace.json`。
- 最后在 Codex 的 `/plugins` 中安装或刷新 `rdc-debugger`。

补充检查：

- Claude Code 平台若仍保留 `.claude/settings.json` 的 `agent = "rdc-debugger"`，它只能承担 host bootstrap / orchestrator 语义；正常用户入口仍然是 `rdc-debugger`。

## 6. 当前口径

- `debugger` 是当前唯一进入首发 GA 规划的 framework。
- `analyzer` / `optimizer` 仍是 `incubating`。
- 当前 `Tools` snapshot 应与 `tool_count` 对齐，并包含 `rd.vfs.*` 导航层、`rd.session.*`、`rd.core.*` discovery/observability 以及 bounded event-tree 语义。
- `rd.vfs.*` 只按 read-only navigation 解释；`tabular/tsv` 只按 projection 解释，不构成独立能力面。
- 所有 `debugger` 平台当前都应提供 `rdc-debugger` 作为 public main skill；`rdc-debugger` 只承担 orchestration，不再承担 formal user entry。
- 当前已实测的是 package-local `tools/` + local-first 工具链，以及 Android remote-only 的 daemon / `MCP` 最小 bootstrap 主链。
- 如需快速复核 remote-only 路径，优先执行 `python tools/scripts/tool_contract_remote_smoke.py --rdc "<sample.rdc>" --transport daemon|mcp`。
- Android remote-only 的完整兼容统计不得继续引用陈旧固定值；是否收敛应以最新 smoke 报告为准。
